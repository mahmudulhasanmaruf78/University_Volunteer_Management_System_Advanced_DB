## 📖 Project Overview

The **University Volunteer Management System** is a database-driven application designed to simplify and automate volunteer management activities within a university. The system centralizes volunteer, coordinator, event, participation, certificate, feedback, and meal management into a structured relational database, reducing manual work and improving data accuracy.

This project was developed as part of the **Advanced Database Management System** course at **American International University-Bangladesh (AIUB)** using **Oracle SQL** and **PL/SQL** concepts.

---

## ✨ Key Features

- 👥 Volunteer Registration & Management
- 📅 Event Creation and Management
- 🧑‍💼 Coordinator Management
- ✅ Volunteer Participation Tracking
- 💬 Feedback & Rating System
- 🏆 Certificate Management
- 🍽️ Meal Management
- 🔐 User Authentication (Volunteer & Coordinator)
- 📊 Admin Dashboard Operations

---

## 🗄️ Database Design

The project follows proper Database Management System (DBMS) design principles, including:

- Entity Relationship (ER) Diagram
- Database Normalization (up to Third Normal Form)
- Relational Schema Design
- SQL Table Creation with Constraints
- Sample Data Insertion
- Primary & Foreign Key Relationships

Core database entities include:

- Volunteer
- Coordinator
- Event
- Participation
- Feedback
- Certificate
- Meal
